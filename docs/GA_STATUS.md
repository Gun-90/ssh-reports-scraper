## 서버 부하 감소

```
전환 전: 29개사 × 30분 간격 = 일 1,392회
전환 후:
  평시:  8개사 × 30분 간격 × 20시간 = 320회
  풀스크래핑: 29개사 × 4회/일 = 116회
  GA 호출: 21개사 × 10회/일(평균) = 210회
  ─────────────────────────────────
  서버 직접: 436회/일 (69%↓)
  GA 분산:  210회/일
```

서버 HTTP 호출 **69% 감소**. GA가 GitHub Actions runner IP로 분산 처리.

## 듀얼모드 (GA 장애 대비)

```
KST 1시 / 7시 / 13시 / 21시 → 서버에서 GA 포함 전체 29개사 스크래핑
그 외 시간 → 서버는 미전환 8개사만, GA는 매시간 standalone 실행
```

GA 장애 시에도 최대 6시간 이내 서버 fallback으로 데이터 공백 방지.
ON CONFLICT upsert로 중복 자동 제거.
